# Support

## Getting Help

Baton is open source and community-driven.

For support:

1. Read `README.md`, `BATON_v3.1.md`, and `skills/` first.
2. Search existing GitHub issues.
3. Open a new issue with complete context.

## What to Include in Support Requests

- What you are trying to do
- What happened vs expected behavior
- Relevant file paths and error output
- Tool/environment (Claude Code, Cursor, Codex, etc.)
- Steps to reproduce

## Issue Types

- `bug`: Something is broken
- `docs`: Documentation improvement
- `enhancement`: Improvement request
- `question`: Usage clarification

## Security Issues

For security-related concerns, use `SECURITY.md` and avoid posting sensitive details publicly.

## Expectations

- Maintainer and community responses are best-effort.
- Clear, reproducible reports are prioritized.
